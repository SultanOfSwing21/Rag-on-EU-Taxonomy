# Natural User Queries Benchmark

Generated from `chunks.jsonl`.

## Purpose

This dataset is intended as a secondary retrieval benchmark with more natural user-style questions than the original FAQ titles.

It contains:
- **648 simple questions**: 2 natural questions for each of the 324 FAQ chunks.
- **100 complex multi-hop questions**: each mapped to 2 semantically related FAQ chunks.
- **748 total questions**.
- **324 / 324 chunks covered**.
- **748 unique question strings**.

## Schema

Each line is JSONL:

```json
{"question": "...", "expected_chunk_ids": ["faq-0001"], "difficulty": "simple", "query_type": "natural_simple", "persona": "CFO"}
```

For complex questions:

```json
{"question": "...", "expected_chunk_ids": ["faq-0001", "faq-0002"], "difficulty": "complex", "query_type": "natural_multihop", "persona": "auditor"}
```

## Generation method

Simple questions were generated by rewriting each FAQ title into two user-oriented formulations using rotating personas such as sustainability officer, CFO, ESG consultant, auditor, asset manager, compliance officer, reporting team, operations manager, bank analyst, investor relations, real estate manager, and legal counsel.

Complex questions were generated by:
1. Building TF-IDF vectors from each chunk's question and answer.
2. Computing cosine similarity between chunks.
3. Selecting 100 high-similarity pairs while limiting overuse of individual chunks.
4. Generating scenario-based questions requiring both expected chunks.

## Recommended use

Use this file as a **naturalised synthetic benchmark**, alongside the deterministic golden dataset.

Suggested metrics:
- Recall@1 / Recall@3 / Recall@5
- MRR
- multi-hop hit rate: whether all expected chunks appear in top-k

## Limitations

- This is synthetic, not collected from real users.
- The simple questions are naturalised but still derived from FAQ titles.
- Multi-hop relevance is based on TF-IDF similarity, not manual legal annotation.
- A small manual spot-check is recommended before presenting final results.
